## Основы тестирования

### Что такое тестирование программного обеспечения?
Процесс проверки соответствия программного продукта требованиям и ожиданиям пользователей.

### Какие основные цели тестирования?
- Нахождение дефектов
- Повышение качества продукта
- Обеспечение уверенности в его работоспособности

### Что такое баг (дефект)?
Отклонение фактического результата от ожидаемого, описанного в требованиях.

### Что такое тест-кейс?
Документ, описывающий шаги для проверки определенной функциональности.

### Что такое тест-план?
Документ, описывающий подход к тестированию, включая цели, объем, стратегию и ресурсы.

### Что такое тест-дизайн?
Процесс создания тест-кейсов на основе анализа требований и рисков.

### Виды тестирования
- Функциональное тестирование
- Нефункциональное тестирование
- Регрессионное тестирование
- Smoke-тестирование
- Exploratory testing
- Тестирование черного ящика
- Тестирование белого ящика
- Тестирование серого ящика
- Позитивное тестирование
- Негативное тестирование

## Техники тест-дизайна

### Эквивалентное разделение (Equivalence Partitioning)
Разделение входных данных на классы эквивалентности, где данные в одном классе обрабатываются одинаково.

### Анализ граничных значений (Boundary Value Analysis)
Тестирование на границах диапазонов входных данных.

### Таблица решений (Decision Table)
Таблица, описывающая возможные комбинации входных данных и соответствующие результаты.

### Pairwise Testing
Техника, которая проверяет все возможные пары входных данных для уменьшения количества тест-кейсов.

### State Transition Testing
Тестирование переходов между состояниями системы.

### Use Case Testing
Тестирование на основе сценариев использования продукта.

## Клиент-серверное взаимодействие

### Что такое клиент-серверная архитектура?
Архитектура, при которой клиент запрашивает данные у сервера, а сервер их обрабатывает и возвращает.

### HTTP
- Протокол передачи гипертекста
- Методы: GET, POST, PUT, DELETE, PATCH, OPTIONS, HEAD
- Статусные коды (200 — OK, 404 — Not Found и т.д.)

### API
- REST API: архитектурный стиль для создания веб-сервисов
- SOAP API: протокол для обмена структурированными сообщениями
- Форматы данных: JSON, XML

## Базы данных и SQL

### Основные SQL команды
```sql
SELECT * FROM <имя_таблицы>;
INSERT INTO <имя_таблицы> (столбцы) VALUES (значения);
UPDATE <имя_таблицы> SET <столбец> = <значение> WHERE <условие>;
DELETE FROM <имя_таблицы> WHERE <условие>;
```

### Ключевые концепции
- PRIMARY KEY: уникальный идентификатор записи
- FOREIGN KEY: связь между таблицами
- JOIN: объединение данных из нескольких таблиц

## Git

### Основные концепции
- Репозиторий: хранилище кода
- Ветка: независимая линия разработки
- Коммит: сохранение изменений
- Push: отправка изменений на сервер
- Pull: получение изменений с сервера

### Рабочий процесс
1. Инициализация репозитория: `git init`
2. Клонирование: `git clone <URL>`
3. Добавление файлов: `git add`
4. Создание коммита: `git commit -m "сообщение"`
5. Отправка изменений: `git push`
6. Получение изменений: `git pull`

### Управление ветками
- Создание: `git branch <имя>`
- Переключение: `git checkout <имя>`
- Создание и переключение: `git checkout -b <имя>`
- Слияние: `git merge <ветка>`

### Полезные команды
```bash
git status  # проверка статуса
git log     # просмотр истории коммитов
git diff    # просмотр изменений
git reset   # отмена изменений
git stash   # временное сохранение изменений
```

## Методологии тестирования

### Waterfall (Водопадная модель)
- Последовательный процесс разработки
- Тестирование начинается после разработки
- Минимальная гибкость
- Подробная документация

### Agile (Гибкая методология)
- Итеративный подход
- Раннее тестирование
- Гибкость к изменениям
- Частые релизы

### V-Model
- Параллельная разработка и тестирование
- Четкое соответствие этапов разработки и тестирования
- Раннее планирование тестирования
- Подробная документация
